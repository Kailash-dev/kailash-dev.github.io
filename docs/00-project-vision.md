# Project Vision

## Purpose

This website is a **client acquisition platform** for an international software engineering consultancy. It is not a developer portfolio, a resume site, or a project gallery.

The site exists to:

1. Build trust with founders, startups, SMBs, and agencies
2. Demonstrate capability through case studies and process clarity
3. Convert qualified visitors into discovery calls

## Positioning

**Product Engineering Partner** — a senior independent engineer who helps founders and growing teams design, build, and ship production-ready software.

### Target Audience (Primary)

- Non-technical SaaS founders building MVPs
- Technical founders needing reliable engineering bandwidth
- Agencies seeking a dependable engineering partner

### Geographic Focus

USA, Canada, UK, Europe, Australia

## Design Philosophy

The experience should feel closer to **Stripe, Linear, Vercel, and Apple** than a typical developer portfolio.

| Principle           | Application                           |
| ------------------- | ------------------------------------- |
| Minimal             | Every element earns its place         |
| Premium             | Restraint over decoration             |
| Timeless            | Avoid trends that date quickly        |
| Trust-first         | Copy and structure reduce hiring risk |
| Conversion-oriented | Clear path to discovery call          |

## What We Avoid

- Technology logo walls
- Skill meters and progress bars
- Animated code backgrounds
- Typing animations
- Generic superlatives ("passionate", "innovative")
- Resume-style layouts
- Placeholder or temporary code

## Success Metrics

| Timeframe  | Visitor should think...                            | Action                  |
| ---------- | -------------------------------------------------- | ----------------------- |
| 5 seconds  | "This person builds products for founders like me" | Continue reading        |
| 30 seconds | "There's a clear process — lower risk"             | Explore Work or Process |
| 2 minutes  | "I trust this person enough to book a call"        | Book discovery call     |

## Technical Quality Bar

- Lighthouse: 100 across Performance, Accessibility, Best Practices, SEO
- Strict TypeScript
- Server Components by default
- Accessible, semantic HTML
- Production-grade from day one

## Long-term Vision (3 Years)

Evolve from senior independent partner to recognized boutique product engineering practice with inbound-led client acquisition, productized MVP offerings, and a library of case studies with measurable outcomes.
